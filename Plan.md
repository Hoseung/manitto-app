# 마니또 앱 - 로그인 로직 설계 (Plan.md)

## 개요
마니또(Secret Santa) 매칭 앱에서의 '로그인' 시스템 설계 문서입니다.

## 로그인 방식: 이름 기반 간이 인증

### 설계 원칙
- 별도의 비밀번호나 회원가입 과정 없이, **등록된 이름만으로 인증**합니다.
- 보안보다 **편의성**을 우선시합니다 (소규모 그룹 내부 사용 목적).
- 각 참가자는 **자신의 마니또만** 확인할 수 있습니다.

### 인증 흐름

```
[사용자] --> 그룹 선택 --> 이름 입력 --> [서버 검증] --> 마니또 표시
```

1. **그룹 선택**: 매칭이 완료된 그룹 목록에서 자신이 속한 그룹을 선택
2. **이름 입력**: 관리자가 등록한 정확한 이름을 입력
3. **서버 검증**: 
   - 해당 그룹에 입력된 이름이 존재하는지 확인
   - 매칭이 완료된 그룹인지 확인
4. **결과 반환**: 해당 참가자에게 배정된 마니또 이름만 반환

### API 설계

#### POST `/api/groups/:id/reveal`
- **요청**: `{ name: string }`
- **응답 (성공)**: `{ manitto: string }`
- **응답 (실패)**: `{ message: string }` (404 또는 400)

### 보안 고려사항
- 이름 정확 일치 검증 (공백 trim 처리)
- 매칭되지 않은 그룹에 대한 접근 차단
- 서버에서 매칭 결과를 반환 (클라이언트에 전체 매칭 정보 노출 안 함)
- 각 요청마다 자신의 마니또만 반환

### 매칭 알고리즘
- **단방향 순환 매칭**: 참가자 목록을 셔플한 후 순환적으로 연결
- 예: [A, C, B] 셔플 결과 → A→C, C→B, B→A
- 모든 참가자가 정확히 한 명에게 주고, 한 명에게 받음

### 향후 확장 가능성
- 비밀번호 추가로 보안 강화
- 세션 기반 인증으로 반복 로그인 방지
- 관리자 전용 비밀번호 추가
- 매칭 히스토리 저장
